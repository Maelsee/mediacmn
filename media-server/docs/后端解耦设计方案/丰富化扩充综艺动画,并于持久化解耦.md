# 丰富化扩充综艺与动画，并与持久化解耦（设计方案）

## 背景与目标
- 目标：在不破坏现有电影/剧集统一接口与流程的前提下，扩展刮削契约以完整承载“综艺”和“动画”的特征；同时解耦丰富化与持久化，将持久化写库纳入任务队列异步执行。
- 现状：
  - 契约模型定义于 `services/scraper/base.py`（movie/series/season/episode 结构）。
  - 统一搜索与类型校正已实现，可直接复用（`services/scraper/manager.py:296-324`）。
  - TMDB数据源包含综艺类型（Reality）与动画类（Anime）丰富字段（平台/生产状态/主页/集类型等）。
  - 丰富器当前直接持久化（`services/media/metadata_enricher.py:173-190`），侧车本地化已入队；持久化需改为入队异步执行。

## 契约扩展设计

### 新增枚举：SeriesSubtype
- 枚举字符串值（与数据库 `subtype` 一致）：`scripted | reality | animation | documentary | news | talk | other`
- 作用：细分系列类型，承载综艺/动画等标识，避免扩枚举破坏统一方法。

### 扩展 ScraperSeriesDetail（series 层）
- 保持原字段不变，并新增以下可选字段：
  - `subtype: Optional[SeriesSubtype]`
  - `original_language: Optional[str]`
  - `networks: List[str]`（平台名称列表，如 `Tencent Video`、`bilibili`）
  - `languages: List[str]`（系列语言数组）
  - `origin_country: List[str]`（来源国家/地区；与 TMDB 的 `origin_country` 对齐）
- 参考现有字段：`series_id/name/original_name/overview/tagline/status/first_air_date/last_air_date/episode_run_time/number_of_episodes/number_of_seasons/genres/poster_path/backdrop_path/vote_average/vote_count/popularity/provider/provider_url/artworks/credits/external_ids/raw_data`（`services/scraper/base.py:117-141`）

### 扩展 ScraperEpisodeItem（season→episodes 列表项）
- 新增：
  - `episode_type: Optional[str]`（如 `standard/special/...`）
  - `absolute_episode_number: Optional[int]`
- 参考现有字段：`episode_id/episode_number/season_number/name/overview/air_date/runtime/still_path/vote_average/vote_count`（`services/scraper/base.py:144-155`）

### 扩展 ScraperEpisodeDetail（episode 层）
- 新增：
  - `episode_type: Optional[str]`
  - `absolute_episode_number: Optional[int]`
- 保持嵌套：`series: Optional[ScraperSeriesDetail]` 与 `season: Optional[ScraperSeasonDetail]`（`services/scraper/base.py:175-194`）

### 可选扩展：CreditType
- 新增人员类型以表达综艺语义：`HOST`、`GUEST`
- 若不改枚举，可用 `ACTOR` 携带 `role="guest"` 表示嘉宾；推荐增加以提升语义性。

### 插件填充规则（TMDB示例）
- 系列子类型：
  - 若 `tv.type == "Reality"` → `subtype = reality`。
  - 若 `genres` 含“动画/Anime/Animation” → `subtype = animation`。
  - 其余按 `genres` 映射到 `scripted/documentary/news/talk/other`，规则优先级：`type` 高于 `genres`。
- 平台与主页：
  - `networks = [n.name for n in data.networks]`
  - `original_language = data.original_language or None`
- 季/集：
  - `episode_type = tmdb.episode.episode_type`
  - `absolute_episode_number = tmdb.episode.absolute_number`（若提供）
- 嘉宾：
  - `guest_stars` → `ScraperCredit(type=GUEST, name=..., role="guest")`；无扩枚举时 `type=ACTOR, role="guest"`。

## 丰富化与持久化解耦设计

### 新增队列任务类型
- 在 `TaskType` 增加：`PERSIST_METADATA`
- 位置：`services/task/task_queue_service.py:46-57`

### 持久化任务载荷（Task.params）
- 通用字段：
  - `file_id: int`
  - `user_id: int`
  - `contract_type: "movie" | "series" | "episode"`
  - `contract_payload: object`（序列化后的契约对象，见下）
  - `idempotency_key: Optional[str]`（可选，用于外部幂等；内部仍以 `ExternalID` 保证）
  - `version_context: Optional<object>`（绑定版本的上下文，参照数据库 `media_version`）
- `contract_payload` 结构：
  - Movie：
    - `movie_id/title/original_title/original_language/overview/release_date/runtime/tagline/genres[]/poster_path/backdrop_path/vote_average/vote_count/imdb_id/status/belongs_to_collection/popularity/provider/provider_url/artworks[]/credits[]/external_ids[]/origin_country[]/budget/revenue/raw_data`
  - Series（含扩展）：
    - `series_id/name/original_name/original_language/overview/tagline/status/first_air_date/last_air_date/episode_run_time[]/number_of_episodes/number_of_seasons/genres[]/poster_path/backdrop_path/vote_average/vote_count/popularity/provider/provider_url/artworks[]/credits[]/external_ids[]/languages[]/origin_country[]/raw_data`
    - `subtype/networks[]`
  - Episode（含扩展）：
    - `episode_id/episode_number/season_number/name/overview/air_date/runtime/still_path/vote_average/vote_count/provider/provider_url/artworks[]/credits[]/external_ids[]/raw_data`
    - `episode_type/absolute_episode_number`
    - 可选嵌套：`series` 与 `season`（同契约定义）

- `version_context` 结构（可选）：
  - `scope: "movie_single" | "season_group" | "series_group" | "episode_single"`
  - `quality: Optional[str]`
  - `source: Optional[str]`
  - `edition: Optional[str]`
  - `variant_fingerprint: Optional[str]`

### 执行流
1) 丰富器：解析标题→搜索→类型校正→详情获取→构造契约对象；入队 `PERSIST_METADATA`（不直接写库）。
2) 工作者：消费持久化任务→反序列化契约→调用 `persistence_service.apply_metadata(...)` 与 `bind_version(...)`→提交事务→完成任务。
3) 侧车本地化：保持独立任务（`SIDECAR_LOCALIZE`）。

### 幂等与重试
- 幂等主键：`ExternalID(user_id, core_id, source, key)` 与 `MediaCore(title/kind)` 组合逻辑确保重复写入不产生脏数据。
- 重试：继承现有队列机制（`max_retries/retry_delay/timeout`）并在任务结果写入 `TaskResult`（`services/task/task_queue_service.py:255-296`）。

## 实施步骤与迁移策略

1. 契约扩展
   - 修改 `services/scraper/base.py`：增加 `SeriesSubtype`，为 `ScraperSeriesDetail/EpisodeItem/EpisodeDetail` 增加新字段；（可选）扩展 `CreditType`。
   - 全字段均设为 `Optional`，保障向后兼容。

2. 插件填充
   - TMDB 插件：按上文规则填充新字段，尽量完善 `guest_stars/episode_type/absolute_episode_number`。
   - Douban 插件：最小兼容（无法提供时置空）。

3. 模型迁移
   - 为 `SeriesExt/EpisodeExt` 增加对应列（默认 NULL）。
   - 生成迁移脚本并回填旧数据（如 `original_language/networks` 可依据 `raw_data` 衍生）。

4. 持久化映射
   - 扩展 `metadata_persistence_service.py` 将新字段写入扩展表；保留幂等逻辑与展示缓存机制。

5. 队列解耦
   - 在 `TaskType` 增 `PERSIST_METADATA`；实现工作者消费逻辑。
   - 丰富器修改：由直接写库改为入队持久化任务（保留侧车本地化）。

6. 测试与验收
   - 使用 `DEVELOPMENT.md` 提供的样例：
     - 综艺系列：`/tv/{series_id}`（`服务端文档: 3156-3315`）→ 验证 `VARIETY/networks/origin_country` 写入正确。
     - 综艺季与集：`3318-3511`、`3514-3584` → 验证 `episode_type/guest_stars` 映射。
     - 动画系列：`3707-3770` → 验证 `ANIME/networks(bilibili)/episode_run_time≈20`。
   - 回归电影与普通剧集流程，确保零回归失败。

## 验收标准
- 统一方法复用：综艺/动画沿用 `TV_SERIES/TV_SEASON/TV_EPISODE` 统一接口与流程。
- 契约完整性：新增字段完整表达综艺/动画差异；均为 Optional，不破坏现有实现。
- 解耦有效：丰富器不再直接持久化；持久化任务入队并正确落库，支持幂等与重试。
- 映射正确：扩展表写入新字段，展示缓存与外部ID、海报/剧照、演职员保持既有策略。

## 风险与兼容性
- 插件数据源覆盖差异：不同来源可能不提供 `episode_type/absolute_episode_number`；以空值容错。
- 回填策略：旧数据可利用 `raw_data` 衍生 `original_language/networks`，但不做强制。
- 性能：队列解耦后持久化吞吐依赖 Redis 与工作者并发配置；保留优先级与重试机制。
