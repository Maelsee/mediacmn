目标与原则
- 面向多类型扩展：统一承载电影、剧集、综艺、动画、纪录片、特别篇等，避免为新类型频繁改表。
- 解耦与高效并重：核心与扩展分层，通用信息与类型特有信息分离；为常用查询建立明确、可组合的查询键与索引。
- 明确层级建模：系列-季-集采用统一的层级建模，支持无季剧（季可为空）与绝对集序（动漫常见）。
- 多源对齐：以外部源（tmdb、tvdb、imdb、豆瓣等）为“规范化主源”，可切换与合并，具备稳定查找键。
- 用户隔离与幂等：所有核心表包含 `user_id`，跨用户数据全量隔离；唯一约束保证去重与幂等。
- 查询优化：按“用例驱动”的索引策略，满足：电影按 `tmdb_id` 查详情、剧集按 系列-季-集 精确查、综艺按“Returning Series”近播数据查、动漫按绝对集序查。

核心表：media_core
- 字段
  - `id` int 主键
  - `user_id` int 索引
  - `kind` enum 索引（`movie|series|season|episode`）
  - `subtype` enum 索引（`scripted|reality|animation|documentary|news|talk|other`）
  - `title` varchar 索引
  - `sort_title` varchar 索引
  - `original_title` varchar
  - `year` int
  - `plot` text
  - `adult` bool
  - `original_language` varchar
  - `canonical_source` varchar 索引（`tmdb|douban|tvdb|imdb`）
  - `canonical_external_key` varchar 索引（与 `canonical_source` 组合唯一）
  - `canonical_tmdb_id` int 索引
  - `display_rating` float
  - `display_poster_path` varchar
  - `display_backdrop_path` varchar
  - `display_date` datetime（上映/首播）
  - `status` enum（`Released|Ended|Returning Series|In Production|Planned`）
  - `popularity` float
  - `created_at` datetime
  - `updated_at` datetime
- 约束与索引
  - 唯一：`(user_id, canonical_source, canonical_external_key)`
  - 组合：`(user_id, kind, title)`、`(user_id, kind, subtype)`、`(user_id, canonical_tmdb_id)`、`(user_id, status)`、`(user_id, display_date)`



类型扩展表
- movie_detail（电影）
  - `id`, `user_id`, `core_id`(唯一)
  - `tagline`, `collection_id`, `rating`, `release_date`, `poster_path`, `backdrop_path`, `imdb_id`, `runtime_minutes`, `budget`, `revenue`, `spoken_languages[]`, `production_countries[]`, `raw_data(jsonb)`, `nfo_path`
  - 索引：`(user_id, release_date)`、`(user_id, imdb_id)`
- series_detail（剧集/综艺/动画）
  - `id`, `user_id`, `core_id`(唯一)
  - `series_type`(`Scripted|Reality|Animation|Documentary|News|Talk|Other`), `first_air_date`, `last_air_date`, `episode_run_time`, `season_count`, `episode_count`, `status`, `rating`, `poster_path`, `backdrop_path`, `overview`, `languages[]`, `origin_country[]`, `networks[]`, `raw_data(jsonb)`, `nfo_path`
  - 索引：`(user_id, status)`、`(user_id, first_air_date)`
  - 备注：增加 `origin_country[]`（来源国家/地区），与 TMDB `origin_country` 对齐，用于地区筛选与统计。
- season_detail（季）
  - `id`, `user_id`, `core_id`(唯一), `series_core_id`
  - `season_number`, `episode_count`, `air_date`, `runtime`, `rating`, `poster_path`, `overview`, `auto_generated`, `raw_data(jsonb)`, `nfo_path`
  - 唯一：`(user_id, series_core_id, season_number)`
- episode_detail（集）
  - `id`, `user_id`, `core_id`(唯一), `series_core_id`, `season_core_id?`
  - `title`, `season_number`, `episode_number`, `absolute_episode_number`, `air_date`, `runtime`, `rating`, `vote_count`, `episode_type`, `still_path`, `overview`
  - 唯一：`(user_id, series_core_id, season_number, episode_number)`
  - 组合：`(user_id, series_core_id, absolute_episode_number)`

通用关联与资源表
- external_ids（外部ID）
  - `id`, `user_id`, `core_id`, `source`, `key`
  - 唯一：`(user_id, core_id, source)`；组合：`(user_id, source, key)`
- genre
  - `id`, `user_id`, `name`（同用户唯一）
- media_core_genre
  - `id`, `user_id`, `core_id`, `genre_id`；唯一：`(user_id, core_id, genre_id)`
- person
  - `id`, `user_id`, `name`（同用户唯一）, `tmdb_id`, `profile_url`
- credit
  - `id`, `user_id`, `core_id`, `person_id`, `role`(`cast|crew|guest`), `character`, `job`
  - 唯一：`(user_id, core_id, person_id, role, job)`
- artwork
  - `id`, `user_id`, `core_id`, `type`(`poster|backdrop|fanart|banner|cover|folder|still`), `remote_url`, `local_path`, `width`, `height`, `provider`, `language`, `preferred`, `exists_local`, `exists_remote`
- media_version
  - `id`, `user_id`, `core_id`, `tags`, `quality`, `source`, `edition`, `scope`(`movie_single|season_group|series_group|episode_single`), `variant_fingerprint`, `preferred`, `primary_file_asset_id`
- file_asset
  - `id`, `user_id`, `storage_id`, `full_path`（同用户唯一）, `filename`, `relative_path`, `core_id`, `version_id`, `episode_core_id`, `size`, `mimetype`, `video_codec`, `audio_codec`, `resolution`, `duration`, `etag`, `asset_role`(`video|audio|subtitle|nfo|image|other`), `bitrate_kbps`, `hdr`, `audio_channels`, `container`, `asset_fingerprint`, `exists`, `status`(`active|deleted|moved`), `deleted_at`, `created_at`, `updated_at`
- collection
  - `id`(TMDB合集ID), `name`, `poster_path`, `backdrop_path`, `overview`, `created_at`, `updated_at`

类型差异与适配
- 综艺：`series_detail.series_type=Reality`；`in_production=true` 与 `status=Returning Series`；嘉宾用 `credit.role=guest`。
- 动画：使用 `episode_detail.absolute_episode_number`；`season_core_id` 可空；`series_type=Animation`。
- 特别篇与无季剧：`episode_type=special`；默认为 `season_number=0`。

查询路径
- 电影按 TMDB ID：`external_id(user_id, source='tmdb', key=<movie_id>) -> core_id`；或 `media_core.canonical_tmdb_id=<movie_id>`。
- 剧集 系列-季-集：`external_id(series_id) -> series_core_id` + `episode_detail` 唯一键。
- 综艺最近播出集：`series_detail(in_production=true)` + `episode_detail` 按 `air_date` 取最近。
- 动画绝对集序：`episode_detail(series_core_id, absolute_episode_number=<n>)`。

迁移与实现路线
1) Alembic 创建/更新表：`media_core`、`content_relation`、`external_id`、`artwork`(含 `still`)、`genre`/`content_genre_link`、`person`/`credit`、`movie_detail`、`media_version`、`file_asset`、`series_detail`、`season_detail`、`episode_detail`。
2) 将现有数据迁移到新结构：电影 -> `movie_detail`；剧集 -> `series/season/episode_detail`；
3) 刮削管线适配：电影按 `/movie/{id}`；剧集按 `/tv/{id}` + `/tv/{id}/season/{n}` + `/episode`；综艺走 `series_detail`；写入 `external_id`、`artwork(type=poster/backdrop/still)`。
4) 接口只读新表，逐步淘汰旧结构。
5) 索引优化与幂等保障。

排除项
- 不实现 `network`、`company` 及相关关联表。

与现有代码的对齐

字段对照与一致化清单
- 统一命名：
  - `original_language`（替代 `origin_language`）用于核心与系列语言字段。
  - `series_type` 与契约的 `subtype` 对齐（字符串枚举：`scripted|reality|animation|documentary|news|talk|other`）。
  - `networks[]` 与契约 `networks[]` 一致；存储为数组或 JSON。
  - `homepage` 一致；为空则不写。
  - `episode_type` 与契约一致（`standard|special|...`）。
  - `absolute_episode_number` 与契约一致；为动画等提供绝对序。
  - `origin_country[]`（系列层）必选；与契约一致。
- 统一扩展表结构：
  - `series_detail` 承载：`series_type/languages[]/networks[]/origin_country[]/status/rating/episode_run_time/season_count/episode_count/poster_path/backdrop_path/overview`。
  - `episode_detail` 承载：`episode_type/absolute_episode_number/still_path`。
  - `season_detail` 承载：`runtime/episode_count/poster_path/overview/air_date/rating`。

持久化任务载荷参考（与契约一致）
- 通用：`file_id/user_id/contract_type(id)/idempotency_key/version_context`。
- Movie 载荷：包含 `origin_country[]/budget/revenue`。
- Series 载荷：包含 `languages[]/origin_country[]/networks[]/subtype(series_type)`。
- Episode 载荷：包含 `episode_type/absolute_episode_number/still_path`；可嵌套 `series/season` 片段用于快速对齐。
