# 阶段计划（6A）

## Align（对齐）
- 目标：统一“作品层”卡片展示；版本管理规则：
  - 电影以单文件为单位（movie_single）。
  - 剧集以季为单位（season_group），无季剧默认自动生成季1。
- 范围：仅数据库模型与迁移；服务与接口按现有契约最小改动适配。
- 现状模型参考：
  - `MediaCore`（models/media_models.py:36-61）
  - `SeasonExt`（models/media_models.py:117-137）
  - `EpisodeExt`（models/media_models.py:139-161）
  - `FileAsset`（models/media_models.py:164-198）
  - `MediaVersion`（models/media_models.py:63-77）

## Architect（架构）
- 模型改动（新增字段均为可空，保障灰度）：
  - MediaVersion：新增 `scope: str`（movie_single|season_group）、`variant_fingerprint: Optional[str]`、`preferred: bool = False`、`primary_file_asset_id: Optional[int]`（仅电影）。
  - FileAsset：新增 `asset_role: Optional[str]`（video|audio|subtitle|nfo|image|other）、`bitrate_kbps: Optional[int]`、`hdr: Optional[bool]`、`audio_channels: Optional[int]`、`container: Optional[str]`、`asset_fingerprint: Optional[str]`。
  - SeasonExt：新增 `auto_generated: bool = False`（无季剧自动季1标记）。
  - EpisodeExt：`season_core_id` 改为可空（Optional），可选新增 `absolute_episode_number: Optional[int]`。
  - Artwork（可选）：新增 `provider: Optional[str]`、`language: Optional[str]`、`preferred: bool = False`。
- 约束与索引：
  - MediaVersion 唯一约束：`(user_id, core_id, scope, quality, source, edition)` 或以 `variant_fingerprint` 唯一；索引 `(user_id, core_id, scope)`、`(user_id, core_id, preferred)`。
  - FileAsset 索引：`(user_id, episode_core_id, version_id)`、`(user_id, storage_id, full_path)`、`(user_id, asset_fingerprint)`。
  - Season/Episode 索引：`(user_id, series_core_id, season_number)`、`(user_id, season_core_id, episode_number)`。

## Atomize（任务拆分）
1. 新增字段的迁移脚本（DDL、索引与默认值）。
2. 无季剧自动季1生成与回填（构建 tv_season core、SeasonExt、迁移 EpisodeExt.season_core_id）。
3. 版本指纹计算与归属：
   - 电影：为每个视频 `FileAsset` 创建/匹配 `MediaVersion(scope='movie_single')`，写入 `primary_file_asset_id` 与 `FileAsset.version_id`。
   - 季：同季聚合，生成/匹配 `MediaVersion(scope='season_group')`，按集写入 `FileAsset.version_id`。
4. 默认版本 preferred 计算：
   - 电影：按质量/来源优先级。
   - 季：按覆盖率（版本下集数/总集数）+质量优先。
5. 查询适配：列表仅返回 `MediaCore.kind in ('movie','tv_series')`；详情与版本查询保持分层。
6. 验证与回滚策略：影子字段比对、采样校验、性能基准。

## Approve（门控）
- 数据一致性检查清单：
  - 无季剧 → 季1生成率 100%，集关联完整；
  - 电影版本一对一文件绑定；
  - 季版本覆盖率计算正确、preferred 唯一；
  - 列表与详情查询性能不下降（索引生效）。

## Automate（实施步骤）
- 迁移顺序：
  1) 添加所有新字段与索引（不写数据）。
  2) 运行“无季剧迁移”任务：构建季1与回填 EpisodeExt。
  3) 运行“版本归属”任务：计算 `variant_fingerprint` / `asset_fingerprint` 并回填 `MediaVersion` 与 `FileAsset.version_id`。
  4) 计算并写入 `preferred`（电影、季）。
  5) 冒烟测试：列表与详情端点。
- 风险缓解：分批迁移（分页）、事务与断点续跑、写保护（只追加不删除）。

## Assess（评估）
- 验收指标：
  - 列表卡片统一且正确；
  - 详情页版本切换可用；
  - 无季剧按季1呈现；
  - 迁移后无主键/外键/唯一约束冲突；
  - 大库性能基线满足要求（分页 200 时 < 300ms）。
- 文档更新：在 `docs/media_card_architecture_design.md` 补充“版本作用域与默认季1”的说明。

## 依赖与不改动项
- 保持 `ExternalID` 与现有抓取管线不变；仅新增字段承载版本与技术参数。
- API 契约维持既有路径，新增筛选参数按需灰度开放。
