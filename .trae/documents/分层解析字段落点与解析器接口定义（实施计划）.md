# 字段落点映射

## Light 输出字段 → 持久化位置
- `filename_raw` → `FileAsset.filename`（models/media_models.py:184）
- `filename_normalized` → `FileAsset.filename` 的规范化值（不额外字段，作为入库前处理）
- `extension` → `FileAsset.mimetype` 或从 `filename` 推导（models/media_models.py:193）
- `media_type_coarse` → `FileAsset.mimetype` 前缀归类（video|audio|image|other）
- `season_hint` → `FileAsset` 轻量快照字段（建议：`season_hint`，若不新增则临时放任务 `params`）
- `episode_hint` → 同上（建议：`episode_hint`）
- `year_hint` → 同上（建议：`year_hint`）
- `quality_tags` → `MediaVersion.tags` 在 Deep 聚类时使用（models/media_models.py:73）
- `parser_version` → 写入任务 `params` 并作为缓存键一部分
- `parser_confidence` → 写入任务 `params`，用于 Deep 重解析判定
- `name_hash` → `FileAsset.asset_fingerprint` 的候选维度之一（models/media_models.py:198）

## Deep 输出字段 → 持久化位置
- `title`、`original_title`、`year` → `MediaCore`（models/media_models.py:44-49）
- `season_number`、`episode_number` → `SeasonExt.season_number`、`EpisodeExt.episode_number`（models/media_models.py:137, 160）
- `edition_tags`、`resolution_tags` → `MediaVersion.edition`、`MediaVersion.quality`（models/media_models.py:74-76）
- `aliases` → 可落审计表或缓存，不必入主表
- `parser_version_final`、`confidence_final`、`parse_trace` → 审计日志或审计表（避免污染主表）

## 版本归属与文件绑定
- 电影版本（单文件）：`MediaVersion.scope='movie_single'`，`MediaVersion.primary_file_asset_id=FileAsset.id`（models/media_models.py:63-77）
- 季版本（覆盖多集）：`MediaVersion.scope='season_group'`，每集 `FileAsset.version_id=该版本id`（models/media_models.py:188）
- 无季剧：自动生成季1（`SeasonExt.auto_generated=True`），按季版本逻辑归属（models/media_models.py:144）

# 缓存与失效
- 键：`file_id:parser_version`
- Light TTL 长，Deep TTL 短
- 失效：文件名/路径/`etag` 变化；解析器 MAJOR 版本升级；语言或插件策略版本变化
- 重处理：优先在 Deep 阶段触发；Light 沿用快照

# 指标与审计
- 指标：命中率、一致率、重解析触发率与收益、刮削成功率相关性
- 审计：`parse_trace` 写日志或审计表；维护版本说明文档
