# 目标
- 统一解析器仅实现一套逻辑，提供两种模式：Light（扫描阶段）与 Deep（刮削阶段）。
- 扫描产出“解析快照”，刮削在快照基础上做“深度解析+重解析校验”。
- 建立版本/置信度/追踪体系，支持演进、可观测与高性能。

## 范围与约束
- 范围：解析器模块、扫描引擎与刮削管线的集成、缓存与失效策略、监控与测试。
- 不改动业务外部接口；必要的字段读写与日志落点由实现时对接现有模型。

## 解析器设计
### 统一接口
- `FilenameParser.parse(input: ParseInput, mode: "light"|"deep") -> ParseOutput`
- `ParseInput`：`filename_raw`, `parent_hint`（目录名/上级），`language_pref`, `site_plugins`（tmdb/douban 等），`existing_snapshot`（可选）
- `ParseOutput`：统一结构，随模式填充不同层级字段（详见下）

### Light 模式（扫描阶段）
- 目标：高吞吐粗分类与去噪，避免语言/站点耦合与复杂推断。
- 规则集：
  - Unicode 规范化、控制字符清除、非法文件字符替换
  - 扩展名识别：video|audio|image|other
  - 模式早停提取：`SxxExx`、中文“第X季/第Y集”、`1x01`、`(YYYY)` 等
  - 质量标签识别：`4K/1080p/HDR/WEBRip/BluRay`（仅标注，不参与语义决策）
- 输出字段：
  - `filename_raw`, `filename_normalized`, `extension`, `media_type_coarse`
  - `season_hint`, `episode_hint`, `year_hint`
  - `quality_tags`
  - `parser_version`, `parser_confidence`（基于规则命中数/冲突数的评分）
  - `name_hash`（规范化标题哈希，用于分组与去重）
- 写入：持久化为“解析快照”，供队列任务与 Deep 模式复用。

### Deep 模式（刮削阶段）
- 目标：面向检索/匹配/归档的精确语义解析，与站点插件协同（TMDB/豆瓣）。
- 输入：优先使用 Light 快照；若 `parser_version` 不匹配或 `parser_confidence` 低于阈值则重解析。
- 规则集：
  - 别名/本地化映射、语言优先策略（中文优先、英文回退）
  - 高级分隔符/后缀清理、集合/剧场版/特典等标签处理
  - 季/集语义对齐：中文数字、绝对集序、番剧/国产剧特殊格式
  - 站点校准：标题/年份/季集与外部 ID 一致性校验
- 输出字段：
  - `title`, `original_title`, `year`, `season_number`, `episode_number`, `edition_tags`, `resolution_tags`, `aliases`
  - `parser_version_final`, `confidence_final`, `parse_trace`（命中与冲突明细，用于审计）
- 写入：更新作品/季/集的最终解析可观测字段；必要时输出审计日志或审计表。

## 集成点
### 扫描引擎（Light）
- 在 `UnifiedScanEngine` 的文件发现流程中调用 Light 模式。
- 将快照写入文件实体的快照字段，并附带到队列任务 `params` 作为输入种子。
- 性能预算：单文件解析 < 2ms；批量时支持协程并发与速率限制。

### 元数据刮削（Deep）
- 在 `metadata_enricher` 中获取快照并做版本/阈值校验，必要时重解析。
- 融合站点插件结果，产出最终语义并落库（作品/季/集层级）；记录 `parse_trace`。
- 失败时回退到 `filename_raw` 的最小可用路径，继续刮削匹配。

## 缓存与失效策略
- 键：`file_id + parser_version`；内存/Redis 两级缓存。
- TTL：Light 快照长期缓存；Deep 解析短期缓存（结合站点限流）。
- 失效触发：
  - 文件名/路径变化、存储 `etag` 变化
  - 解析器主版本升级（强制重解析）
  - 语言优先或插件策略版本变化
- 重处理策略：优先在 Deep 阶段触发；Light 阶段沿用快照以保吞吐。

## 版本与置信度
- 版本采用语义化：`MAJOR.MINOR.PATCH`
  - MAJOR：规则变更影响结果一致性 → 强制重解析
  - MINOR：新增规则/标签 → 深度解析优先使用，轻量可不重算
  - PATCH：修复 → 仅低置信样本或命中冲突样本重算
- 置信度：0–100，按命中规则权重与冲突扣分计算；设阈值如 60。

## 监控与审计
- 指标：
  - Light 命中率与置信度分布
  - Light→Deep 一致率（标题/年份/季集）
  - 重解析触发率与修正比例
  - 刮削成功率与置信度相关性
- 审计：持久化 `parse_trace`（JSON）到日志或审计表；维护“解析器版本说明”。

## 测试与验证
- 单元测试：
  - 正常样例、中文季集样例、复杂命名（合集/特典）、噪声标签（4K/HDR）
- 回归集：Golden 文件名 → 期望输出比对（Light/Deep）
- 压测：万级文件批量 Light 解析，阈值与缓存命中评估。

## 实施分阶段
- Phase 1：实现统一解析器与 Light 模式；扫描集成与快照落点。
- Phase 2：实现 Deep 模式与站点插件校准；置信度与重解析策略；`parse_trace` 输出。
- Phase 3：缓存与失效策略上线；监控指标与可视化；灰度参数。
- Phase 4：规则版本发布管理与文档化；低置信度样本自动回看队列。

## 验收标准
- Light 与 Deep 统一契约、单一实现；双模式输出清晰一致。
- 扫描延迟稳定、刮削准确率提升；重解析触发可控。
- 指标齐备，可观测与可审计；版本升级可控且安全。
