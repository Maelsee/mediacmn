# 6A工作流：季级版本管理数据库与流程方案（不落地）

## 背景澄清
- 需求：同一剧集的同一季存在多个文件来源/清晰度（如 1080p 与 4K），这些资源不重复、都保留；**版本只在季级管理**，**集级不设版本维度**。
- 目标：对齐前端展示（季的不同版本列表、每季的集范围与覆盖情况），与分层解析策略兼容。

## 数据模型调整（最小必要）

### MediaVersion（承载“季的文件版本”）
- 语义：代表某一季的一种文件变体（质量/来源/版别）。
- 字段：
  - 保留：`user_id, core_id, tags, quality, source, edition`
  - 新增：`variant_fingerprint: Optional[str]`（quality+source+edition+编码要素规范化指纹）
  - 新增：`preferred: bool = False`（该季的默认版本）
- 约束：将唯一约束调整为或增加 `Unique(user_id, core_id, quality, source, edition)` 或以 `variant_fingerprint` 唯一。
- 关联规则：`core_id` 指向 **该季的 MediaCore(kind='tv_season')**。

### FileAsset（承载每集的具体文件）
- 保留：`episode_core_id`（指向该集的 MediaCore）、`version_id`（指向季的 MediaVersion）
- 建议新增技术字段（便于聚类与排序）：
  - `bitrate_kbps: Optional[int]`
  - `hdr: Optional[bool]`（HDR10/DV）
  - `audio_channels: Optional[int]`
  - `container: Optional[str]`（MKV/MP4）
  - `asset_fingerprint: Optional[str]`（技术参数指纹，快速去重）
- 字段使用约定：
  - **剧集文件**：`episode_core_id = 该集 core`；`version_id = 该季的 MediaVersion`；`core_id` 可为空或指向季 core（按现状沿用）。

### SeasonExt / EpisodeExt（规范化来源增强）
- SeasonExt：新增 `canonical_source: Optional[str]`、`canonical_external_key: Optional[str]`（与选中版本的来源对齐，可选）
- EpisodeExt：保持现状；不引入集级版本维度。

### Artwork（首选封面）
- 新增：`provider: Optional[str]`、`language: Optional[str]`、`preferred: bool = False`，支持与选中季版本对齐的封面选择（可选）。

## 解析与归属流程（与分层解析对齐）

### 扫描阶段（轻量）
- 为每个文件写入 FileAsset，记录轻量快照（规范化名、扩展名、基本技术参数）。
- 不做版本归属，仅做季/集粗分类。

### 刮削阶段（深度）
1. **归一化：同季聚合**
   - 基于外部ID/高置信度匹配，将文件归到季级 MediaCore。
2. **版本识别：季级聚类**
   - 在同季内，按 `quality + source + edition + 编码要素` 聚类，生成或查找 `MediaVersion`（季级）。
   - 计算 `variant_fingerprint`，确保唯一。
3. **文件归属：集关联版本**
   - 为每个 Episode 的 FileAsset，设置 `version_id = 该季的 MediaVersion`。
   - 若同一季存在多版本且某集在某版本缺失，则该集的 FileAsset 指向另一个版本；允许“版本覆盖不完整”。
4. **默认版本选择**
   - 规则：优先选择覆盖率最高的版本（包含集数最多），若覆盖率相同则选择质量/来源更优的版本，标记 `preferred = True`。
   - 该选择用于前端季卡片的“默认播放版本”。

## 前端展示与查询
- **剧集卡片**：
  - 季列表：来自 `SeasonExt`；每季显示默认版本（`preferred`）的封面与质量标签。
  - 版本列表：查询 `MediaVersion(core_id=季 core_id)`；展示质量/来源/版别、覆盖率（版本下已关联的集数 / 该季总集数）。
- **季详情**：
  - 集列表：来自 `EpisodeExt`；每集的可用文件通过 `FileAsset(episode_core_id)` 查询；如有多个版本，按所选季版本优先，回退至其他版本。

## 索引与性能建议
- MediaVersion：`(user_id, core_id, preferred)`、`(user_id, core_id, quality, source)`、`(user_id, core_id, variant_fingerprint)`
- FileAsset：`(user_id, episode_core_id, version_id)`、`(user_id, storage_id, full_path)`、`(user_id, asset_fingerprint)`
- SeasonExt/EpisodeExt：`(user_id, series_core_id, season_number)`、`(user_id, season_core_id, episode_number)`

## 迁移策略（不落地）
- Phase 1：为 MediaVersion 增加 `variant_fingerprint, preferred`；为 FileAsset 增加技术字段与 `asset_fingerprint`。
- Phase 2：编写聚类与归属逻辑（离线或增量）：
  - 按季聚合同质文件为版本；补齐 FileAsset.version_id；计算版本覆盖率并选出 `preferred`。
- Phase 3：前端查询改造，支持季版本列表与默认版本切换；集列表按所选版本优先展示。

## 验收标准
- 同季的多个来源/清晰度在前端作为“季版本”可浏览与切换；默认版本合理。
- 集级不设版本维度，但每集文件正确指向季版本；缺集回退策略明确。
- 解析器深度阶段稳定完成版本聚类与归属；索引满足页面性能需求。

## 结论
- 以季为单位管理版本（MediaVersion 关联季 core），集级不引入版本维度；通过 FileAsset.version_id 将集级文件归属到季版本，实现前端“季的不同版本”展示与播放选择，同时保持模型与解析流程最小改动、清晰一致。
