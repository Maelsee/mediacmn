## 根因推断
- 高概率未“加载并启用”任何插件：`sorted_plugins` 仅包含“已启用且已加载”的插件（`manager.py:281-285`），否则 `tasks` 为空。
- 次要可能：即使启用 TMDB，但仅配置了 `v4 token`、缺少传统 `TMDB_API_KEY`，TMDB v3 搜索端点会返回非 200，代码在 `tmdb.py:201-207` 转为空结果。

## 先验证步骤（不改代码）
1. 检查插件状态
- 调用 `GET /api/scraper/plugins`，确认每个插件的 `enabled` 与 `loaded` 字段。若均为 `false`，即为空结果的直接原因。

2. 正确加载并启用 TMDB（推荐）
- `POST /api/scraper/plugins/tmdb/configure`
  - 请求体：`{"config": {"api_key": "<你的TMDB_API_KEY>", "language": "zh-CN"}}`
- `POST /api/scraper/plugins/tmdb/enable`
- （可选）`POST /api/scraper/plugins/tmdb/test` 期待返回成功。

3. 可选启用 Douban
- `POST /api/scraper/plugins/douban/enable`
- 若 Douban 需 Cookie/密钥，先行 `configure`。

4. 复测搜索
- `POST /api/scraper/search`，请求体：`{"title": "七月与安生", "year": 2016, "media_type": "movie", "language": "zh-CN"}`
- 观察日志：应出现
  - `TMDB搜索: ...`（`tmdb.py:167`）
  - `TMDB请求URL/响应状态`（`tmdb.py:196-203`）
  - `TMDB原始结果数/搜索完成`（`tmdb.py:210-221`）
- 返回不应为空；若仍为空，检查 `tmdb.py:201-207` 是否打印非 200 与错误响应，通常是 API Key 认证问题。

## 判定标准
- `GET /api/scraper/plugins` 显示至少一个插件 `enabled=true` 且 `loaded=true`。
- 搜索日志出现插件级别日志，且响应状态为 200。
- 返回 `count > 0`，包含期望条目。
