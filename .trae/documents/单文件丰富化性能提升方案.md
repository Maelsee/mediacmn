# 目标
- 将侧车文件写入改为有界并发上传，并按语言优先与评分筛选艺术作品数量。
- 在 TMDB 详情接口使用 append_to_response 聚合 credits 与 images，减少往返次数。

# 变更范围
- 修改 `services/media/metadata_enricher.py` 的 `_write_sidecar_files`：
  - 读取 `SIDE_CAR_LOCALIZATION_ARTWORK_LIMIT`；筛选出少量优先艺术作品。
  - 使用 `asyncio.Semaphore` 控制并发，`asyncio.gather` 并发上传 `.nfo` 与艺术作品。
- 修改 `services/scraper/tmdb.py` 的 `_get_media_details`：
  - 将 `append_to_response` 扩展为 `external_ids,keywords,credits,images`。
  - 保持当前共享会话与错误处理，兼容无法聚合的端点。

# 验收
- 单文件侧车时间由 ~13s 降至 ~7–9s；上传日志时间戳更密集。
- 刮削阶段往返次数减少；详情获取时间缩短。

- 执行一次 `metadata_fetch` + `sidecar_localize`，对比日志中的开始与完成时间。
- 检查目录仅上传限额内的海报/背景图；`.nfo` 正常。
