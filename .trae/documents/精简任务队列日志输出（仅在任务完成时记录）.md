# 目标
- 减少任务队列（TaskQueueService）日志噪声，将非关键日志降级为 debug，仅在任务完成（complete_task）时输出 info 日志。

# 具体调整范围
- `services/utils/task_queue_service.py`
  - `connect/disconnect`：info → debug（保留 error）。
  - `enqueue_task`：info → debug（保留 error）。
  - `dequeue_task` 获取任务：info → debug（保留 error）。
  - `complete_task`：保留 info（任务完成时输出成功/失败、耗时等）。
  - `cancel_task`：info → debug（保留 warning/error）。
  - `get_queue_stats/cleanup_expired_tasks`：info → debug（保留 error；过期清理可降为 debug）。

# 不变内容
- 错误与警告（error/warning）保持原级别，便于故障定位。
- 执行器/调度器的任务完成日志（成功/失败）保持 info，不属于队列层，但有助于总体观测。

# 验收
- 队列层日志在正常运行下只在 `complete_task` 时输出 info；其他队列事件无 info 噪声。
- 出错场景仍有 error/warning；不影响排障。

# 回滚策略
- 如需恢复详细日志，将相关 debug 恢复为 info 或引入配置开关控制日志级别。
